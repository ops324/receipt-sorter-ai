# アリサ Web版 デプロイ手順書 ＆ iPad実機チェックリスト

iPad Safari で使える状態にするための、迷わない順序の手順書。
所要時間の目安: **10〜15分**（Vercel/GitHub連携済みなら数分）。

## 構成（現行：都度処理・BYOK）
- **純粋な静的SPA**。サーバー無し・データベース無し・ログイン無し（※任意で**パスワードゲート**を有効化でき、URLを知った部外者を締め出せる。サーバー認証ではない → STEP 1.5）。
- OCR（読み取り）は**クライアント本人のAnthropicキーでブラウザから直接**呼ぶ（BYOK）。
- データは**その端末（iPad）のブラウザ内（IndexedDB）**に保存。オーナー側インフラ・課金はゼロ。
- **環境変数の設定は不要。Supabaseも不要。**

> 💡 以前の「Supabase＋サーバー関数＋招待制」構成は廃止しました。過去のenv/招待/schema手順はもう使いません。

| 必要なもの | 入手先 |
|-----------|--------|
| Vercel アカウント | https://vercel.com （GitHub連携が楽） |

## STEP 1. Vercel にデプロイ（静的サイト）

1. https://vercel.com → **Add New… → Project** → このリポジトリを Import。
2. **設定で必ず**:
   - **Root Directory = `web`**（リポジトリ直下ではない。Mac版と同居しているため）
   - Framework Preset: **Vite**（自動検出されるはず）
   - Build Command / Output: 既定（`npm run build` / `dist`）でOK
3. **Environment Variables は不要**（空のままでよい）。
4. **Deploy** を押す。完了するとURL（`https://xxxx.vercel.app`）が発行される。

> 以降は `main` に push するたび Vercel が自動で再デプロイする（中心銘と同じ運用）。

---

## STEP 1.5. パスワードゲート（任意・部外者対策）

公開URLを「このクライアント本人」だけに使わせたい場合に有効化する。**設定しなければゲートは出ず、これまで通り誰でも開ける**（opt-in）。

1. 顧客に渡す**パスワード**（8文字以上を推奨）を決め、リポジトリ直下で生成:
   ```
   node web/scripts/gen-access-code.mjs '決めたパスワード'
   ```
2. 出力された `ACCESS_SALT_B64` / `ACCESS_HASH_B64` を **`web/src/lib/access-gate.ts`** の同名定数に貼り付ける（**パスワードそのものはコミットしない**。埋め込むのはハッシュのみ）。
3. `main` に push → Vercel が自動再デプロイ。以降、URLを開くとパスワード入力画面が出る。
4. 顧客にはパスワードを口頭/別経路で伝える。**正しく入れた端末は以後スキップ**（`localStorage['arisa.access']` に記録）。
5. パスワードを変えたい／流出したら、再生成して定数を差し替え push するだけ（既存端末も自動で再度ゲートに掛かる）。

> ⚠️ これは「URLを偶然知った部外者を気軽に使わせない」簡易ゲート（casual deterrent）。アプリは静的公開のため、技術者はソースを読めば回避し得る。厳密な遮断が要る場合は Vercel のパスワード保護やエッジ認証を検討（本手順の範囲外）。

---

## STEP 2. クライアントの初期設定（あなたが代行推奨）

クライアントのiPadで一度だけ:

1. Safari で発行URLを開く。
2. **ホーム画面に追加**（共有ボタン → ホーム画面に追加）。
   - ⚠️ **重要**: これをしないと iOS Safari は約7日でブラウザ保存（IndexedDB）を消去する。ホーム画面アプリ化すると保存が永続化される。
3. アプリを開き、**「設定」→ Anthropic APIキー** に本人のキー（`sk-ant-...`）を貼って **保存**。
   - キーはこの端末内にのみ保存され、サーバーには送られない。
4. Anthropicコンソールで **Spend Limit** を設定（未設定なら今ここで）。

---

## STEP 3. iPad 実機チェックリスト

iPad の **Safari（またはホーム画面アプリ）** で開き、上から順に確認する。✅が全部付けばiPadで使える状態。

### 3-1. 基本
- [ ] ページが表示される（パスワードゲート**未設定**なら直接インボックスが開く。**設定済み**ならパスワード入力画面が出て、正しいパスワードで解除→以後その端末はスキップ）
- [ ] サイドバー／各画面（インボックス・一覧・集計・案件・ルール・エクスポート・設定）が表示される
- [ ] 「設定」に **Anthropic APIキー入力欄** がある → キーを保存すると「この端末に設定済み」になる

### 3-2. 取込 → OCR（コア機能・最重要）
- [ ] キー未設定で取込すると「APIキーが未設定です」とトーストが出る（＝ガードが効く）
- [ ] キー設定後、インボックスで取込 →「写真を撮る」「ファイルを選択」が出る
- [ ] iPadのカメラで領収書を撮影／選択 → 「処理中」→ しばらくして「**要確認**」または「確定済」に変わる（＝**本人の鍵で読み取り成功**）
- [ ] 読み取り結果（店名・金額・日付）が一覧に出る
- [ ] **複数枚**を一度に選んで取込 → 進捗が進み、全部処理される

### 3-3. 計算（今回の主眼）
- [ ] 「集計」タブで、期間・案件・確定のみで絞り込み、**勘定科目別／支払方法別の合計・税額**が出る
- [ ] 数値がブラウザ内計算のみで即時表示される（追加課金なし）

### 3-4. プレビュー（⚠️ 既知の要確認ポイント）
- [ ] 画像領収書（JPEG/PNG）のプレビューが編集パネルで表示される
- [ ] **PDF領収書のプレビューが表示される**（← `pdfjs` の iOS Safari 動作。崩れたら開発側で対処）

### 3-5. 編集・管理
- [ ] 編集パネルで店名・金額・科目・案件・支払方法を変更 → 保存される
- [ ] 一覧のソート・フィルタ・検索が効く
- [ ] 案件・ルールを作成/編集/削除できる（ルール追加で該当店名に科目が自動付与）
- [ ] 領収書を削除できる
- [ ] **アプリを閉じて開き直してもデータが残っている**（IndexedDB永続化の確認。ホーム画面アプリ化済みが前提）

### 3-6. 未実装（今はNGでOK）
- [ ] エクスポート（CSV/Excel等のファイル書き出し）→ **「準備中」エラー**が現状の正常動作（次段で実装予定。集計は「集計」タブで確認可）

## トラブル時の切り分け

| 症状 | 見るところ |
|------|-----------|
| ビルド失敗 | Vercelの **Root Directory = `web`** 設定漏れ（最頻） |
| 取込で「APIキーが未設定です」 | 「設定」で本人のキーを保存したか |
| OCRが必ず failed | キーの有効性・残高、Spend Limit超過。ブラウザのコンソール（開発）でエラー確認 |
| 数日空けたらデータが消えた | ホーム画面アプリ化していない（Safariの7日消去）。STEP2-2を実施 |
| 別のiPad/ブラウザだとデータが無い | 仕様。データは端末内（IndexedDB）に閉じており同期しない |
| パスワードゲートが出したいのに出ない | `web/src/lib/access-gate.ts` の `ACCESS_HASH_B64`/`ACCESS_SALT_B64` が空。STEP 1.5で生成・貼付して push |
| 顧客がパスワードを忘れた／変えたい | 開発者側で再生成して定数を差し替え push（既存端末も再ゲートされる）。STEP 1.5-5 |
